☆[10.28]小x的日记
描述 Description
  OIer最良好的习惯就是写日记，把每天做的事情记录下来，尤其犯的错误，对算法的理解和总结。每天写一下自己的心得，最后一周你的复习才会有主线。
小x的日记一般都是保密的，小x还有一个爱好：看小说。作业可以不做，小说不能不看。小x的日记一般都会记录下今天做的事情，当然每天看小说的记录也是必须要记录的。
小x的日记格式如下：
10月2日：切水题数量   2 道
10月2日：吃包子　　   4 个
10月2日：小说看到　　 230 章
10月3日：搜索题目 1 道
10月3日：小说看到 232 章
10月4日：小说看到 232 章
……
小x的日记是每天晚上临睡前记录的，所以肯定按照日期顺序给出。小说是必须看的，所以日记里每天都会有小说看到的章节记录。在同一天，不同的项目只会出现一次。
某天，老师在小x的电脑里发现了这个日记，但是在网络传输中，因为学校最近网络掉包率太高，所以日记文件只剩下包含数字的那一列。比如对应上面的例子，只剩下 2 4 230 1 232 232……。根据日记文件的属性，老师算出小x这个日记包含的天数N，现在老师想知道，小x在日记里，每天可能看小说的章节。比如以上这个例子，只需要输出：230 2 0。  即可。
输入格式 Input Format
  第一行两个整数：m和n。m表示一共日记里有m个数据项，n表示日记包含了n天。
接下来m行，每行一个正整数，记录这个事情的数据项Di。  Di<=2^31-1.
输出格式 Output Format
  如果数据中找不到合法的小说记录，输出-1.
否则输出n行，每行一个正整数Xi，表示第i天看了Xi章小说。
如果有多组方案，输出前面的数尽量大的方案（字典序最大）。

样例输出 Sample Output
 
时间限制 Time Limitation
  1s
注释 Hint
  【数据范围】
对于30%的数据，m ≤ 10；
对于70%的数据，m ≤ 1 000；
对于100%的数据，m ≤ 100 000,  n<=m.
